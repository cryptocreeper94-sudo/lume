MARKETING DESCRIPTION
=====================
Lume is the first programming language where artificial intelligence is a
syntax primitive, not a library import. While traditional languages require
SDKs, API wrappers, and boilerplate to integrate AI, Lume treats AI as a
native feature of the language itself. Developers write "ask", "think", and
"generate" as keywords — the same way they write "if" or "for" in any other
language.

But Lume goes further than any language before it. Lume programs are alive.
They monitor their own performance, heal themselves when errors occur,
optimize their own bottlenecks, and evolve their behavior over time. The
self-sustaining runtime — four layers deep — means that a deployed Lume
program can run autonomously, only calling a human when it faces a decision
it genuinely cannot make on its own.

Lume is not just a language. It is the first step toward software that takes
care of itself.

Built by DarkWave Studios. Powered by Trust Layer. Protected by TrustShield.

ARCHITECTURE
============

Language Pipeline:
  .lume source -> Lexer -> Token Stream -> Parser -> AST -> Transpiler -> .js

Self-Sustaining Runtime (4 Layers):
  Layer 1: Self-Monitoring  — Function metrics, AI call stats, memory, dashboards
  Layer 2: Self-Healing     — Retry, circuit breakers, model fallback chains
  Layer 3: Self-Optimizing  — Bottleneck detection, mutation log, rollback
  Layer 4: Self-Evolving    — Pattern learning, cost analysis, evolution decisions

Toolchain:
  - lume run <file>     — Transpile + execute
  - lume build <file>   — Transpile only
  - lume test <file>    — Run test blocks
  - lume fmt <file>     — Format source code
  - lume lint <file>    — Lint with 15 rules + suggested fixes
  - lume repl           — Interactive REPL with multi-line support
  - lume watch <file>   — File watcher with auto-rebuild
  - lume ast <file>     — Print AST
  - lume tokens <file>  — Print token stream
